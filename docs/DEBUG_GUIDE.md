# DEBUG_GUIDE.md

本文件记录项目调试入口。

不要把这里写成问题流水账。具体已解决问题写入 `docs/ERROR_CATALOG.md`。

## 1. 调试入口

- 串口：
- RTT / SWO：
- CAN 工具：
- RTOS trace：
- kernel log：
- crash dump：
- 调试器：
- 台架或硬件连接：

## 2. 日志规则

- 只保留关键片段。
- 保留时间戳、ID、错误码、地址、帧 ID、状态跳变。
- 区分观察事实和假设。
- 不用单行日志直接下结论。

## 3. AI 调试流程

1. 复述现象。
2. 找到疑似归属模块。
3. 标出风险区域。
4. 给出最小检查计划。
5. 未确认归属前，不随意改代码。
