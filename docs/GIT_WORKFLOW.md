# GIT_WORKFLOW.md

本文件记录项目 Git 工作流。真实项目应按团队习惯补充。

## 1. 默认规则

- 大任务完成后，在 `tasks/` 下写日期化任务记录，若项目启用该目录。
- 修复具体问题后，更新 `docs/ERROR_CATALOG.md`。
- 确认或改变工程决策后，更新 `docs/DECISIONS.md`。
- 影响控制算法、观测器、估计器、状态量或命令语义时，更新 `docs/CONTROL_THEORY.md`。

## 2. 提交前检查

- 确认修改范围符合任务。
- 确认没有提交临时 debug 代码。
- 确认没有提交无关格式化。
- 确认必要验证已运行，或原因已说明。
